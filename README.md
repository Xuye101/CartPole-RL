## 🌟 Key Highlights & Optimizations (Online RL)

Rather than blindly tuning hyperparameters, this project focuses on **critical thinking and architectural debugging**.

### 1. The A2C Stability Fixes
A naive implementation of A2C often suffers from early policy collapse. We implemented three core modifications:
*   **Actor-Critic Decoupling:** Separated the shared hidden layers between the Actor and Critic networks to prevent gradient interference.
*   **Orthogonal Initialization:** Applied orthogonal weight initialization to ensure initial action probabilities are near 50/50.
*   **Gentle Reward Scaling:** Replaced standard normalization with a gentle scaling factor (`returns / 100.0`) to eliminate vanishing/exploding gradients in early episodes.

### 2. The PPO Sample Efficiency
*   **Trust Region Updates:** By utilizing `eps_clip = 0.2`, the agent safely re-uses the same batch of transitions for `K_epochs = 4` updates without destroying the current policy.

## 🚀 Getting Started

### Prerequisites
Ensure you have the following installed:
```bash
pip install torch numpy gymnasium
```

### 1. Running Online RL (DQN, A2C, PPO)
We have a unified training script. To run `cartpole_a2c.py`, `cartpole_dqn.py`, or `cartpole_ppo.py`, simply edit the `CURRENT_ALGORITHM` variable inside `train.py`:

```python
# Options: "dqn", "a2c", "ppo"
# ----------------------------------------------------
CURRENT_ALGORITHM = "a2c" 
```
Then run the script:
```bash
python train.py
```

### 2. Running Offline RL (Behavioral Cloning)
To train an agent offline without environment interaction, follow these steps:

**Preparation:**
Ensure you have a fully trained PPO expert model saved at `models/cartpole_ppo.torch`. *(This is generated by running `train.py` with `CURRENT_ALGORITHM = "ppo"`).*

**Step 1: Generate Expert Data**
If you haven't generated the expert data (`expert_data.npz`), run the data generation script. This relies on the PPO model above:
```bash
python generate_data.py
```

**Step 2: Train the Offline Model**
Run the offline training script to train the Behavioral Cloning (BC) model:
```bash
python train_offline.py
```

## 🌟 核心亮点与架构优化 (在线 RL)

本项目拒绝盲目调参，重点展示**对算法瓶颈的批判性思考与架构级调试**。

### 1. 解决 A2C 算法崩溃痛点
原生的 A2C 实现极易在早期陷入策略崩溃。我们进行了三项核心修改：
*   **Actor-Critic 网络解耦：** 剥离了 Actor 和 Critic 共享的隐藏层，防止价值网络的损失干扰策略网络的梯度。
*   **正交权重初始化：** 确保初始状态下智能体向左和向右移动的概率接近 50/50，防止模型过早收敛到次优策略。
*   **温和的奖励缩放：** 摒弃传统的均值方差标准化，改用温和的缩放因子（`returns / 100.0`），彻底消除了早期批次数据导致的梯度爆炸问题。

### 2. PPO 的样本效率优化
*   **信任域截断更新：** 通过设置截断参数 `eps_clip = 0.2` 作为安全锁，智能体可以在同一批经验数据上安全地进行多轮更新（`K_epochs = 4`），提升了样本利用率。
